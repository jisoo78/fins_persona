# Amy Hood Event Matching Evaluation Plan

Date: 2026-07-20

## 목적

기존 1.5.0 객관식 평가는 함정 선택지를 어렵게 만들어 모델 차이를 확인했다. 다만 객관식은 여전히 보기 안에서 정답을 고르는 방식이라, 페르소나가 실제 의사결정 기준을 스스로 불러와 적용하는지 확인하기 어렵다.

이번 평가는 실제 사건과 가상 사건을 매칭하게 만들어, Amy Hood 페르소나가 어떤 과거 의사결정 사례를 근거로 판단하는지 확인하는 것을 목표로 한다.

## 평가 방식

1. 실제로 있었던 Microsoft 의사결정 사건을 기준 라이브러리로 둔다.
2. 실제 사건과 유사하지만 동일하지 않은 가상 사건을 만든다.
3. 페르소나에게 가상 사건을 제시한다.
4. 페르소나는 데이터베이스에서 가장 유사한 실제 사건을 고른다.
5. 선택한 실제 사건의 판단 기준을 가상 사건에 적용해 주관식으로 답한다.
6. 답변은 루브릭으로 채점한다.

## 채점 루브릭

각 문항은 10점 만점으로 채점한다.

- 유사 실제 사건 선택: 0~2점
- 의사결정 기준 유사도: 0~2점
- 근거 사용: 0~2점
- 추측 억제: 0~2점
- 최종 권고 명확성: 0~2점

현재 구현은 1차 자동 채점을 포함한다. 답변에 기대 유사 사건 ID, 판단 기준 키워드, 근거 한계 표현, 확인 필요 항목, 최종 권고가 들어갔는지를 휴리스틱으로 점수화한다. 이 점수는 빠른 비교용이며, 최종 성능 평가는 사람이 보거나 별도 LLM-as-Judge JSON으로 보정할 수 있다.

## 기존 평가와의 차이

1.5.0 객관식 평가는 모델이 함정 선택지를 구분하는지 확인한다.

Event Matching 평가는 모델이 스스로 과거 사건을 불러오고, 그 사건의 판단 기준을 새로운 사건에 적용하는지 확인한다.

따라서 이 평가는 단순 정답률보다 의사결정 기준 복제 여부를 더 직접적으로 볼 수 있다.

## Web UI 연결

B트랙 평가 실행 화면에 Event Matching 패널을 추가했다.

- 평가셋 버전, 실제 사건 수, 문항 수, 완료 실행 수를 표시한다.
- 모델을 선택해서 Event Matching 평가를 실행할 수 있다.
- 실행 결과 목록에서 모델, 실행 ID, 상태, 문항 진행 수, 자동 점수, 완료 시각을 확인할 수 있다.

CLI 실행도 유지한다.

```bash
npm run event-matching:evaluate -- --model=gemma4-12b-local
```
