01 · Executive summary

최종 요약

질문지 2.0.0은 기존 객관식 12/12 만점 포화를 Amy Hood + RAG 조건에서 7/12로 낮췄다. 그러나 통제된 3조건 비교에서는 RAG lift가 -3점, Persona lift가 -2점이었고, 일반 CFO Prompt가 객관식 12/12를 기록했다. 따라서 현재 PoC의 병목은 “모델이 문제를 너무 잘 푼다”보다 일반 CFO 판단과 Amy Hood 고유 판단을 분리하지 못하는 판별 타당성, 그리고 top-1 원문 RAG가 판단을 방해하는 검색·컨텍스트 품질에 있다.

02 · Evaluation design

평가 설계와 비교 가능성

두 종류의 비교를 분리한다. 질문지 1.0.0과 2.0.0의 Before/After는 난이도 강화 이후의 관찰 변화(observed change)를 보여주지만 질문 자체가 달라 인과 추론에는 사용할 수 없다. RAG와 Main Prompt 효과는 동일한 질문지 2.0.0·모델·스냅샷을 공유한 3조건 절제 실험에서만 판단한다.

05 · Hypothesis review

가설, 판정과 근거 논리

부분 지지

H1. 질문지 2.0.0은 만점 포화를 완화한다

Amy Hood + RAG 객관식이 12/12에서 7/12로 하락했다. 다만 질문지가 변경된 Before/After이므로 난이도 상승의 방향은 관찰했지만 효과 크기를 독립적으로 추정할 수 없다.

기각

H2. top-1 RAG는 과거 복원을 향상한다

동일 질문지 2.0.0에서 RAG 포함은 4/7, 미포함은 7/7이었다. 현재 검색·원문 청크·컨텍스트 조합의 RAG lift는 -3이다.

기각

H3. Main Prompt는 홀드아웃 재현성을 높인다

Amy Hood/RAG 없음은 3/5, 일반 CFO는 5/5였다. 현재 Main Prompt의 Persona lift는 -2이며 오히려 H3·H5의 근접 함정에 더 끌렸다.

구현 검증 통과

H4. 역할 분리와 증거 주입 계약이 동작한다

실제 system 역할에는 Persona/Control Prompt만, user 역할에는 질문·선택지·출력 형식과 조건부 RAG 증거(evidence)가 들어간다. 자동 테스트에서 RAG 증거는 첫 실험군 P1-P7의 정확히 7회에만 검출됐다.

응답 계약 문제: 이유는 “사업부별 영향을 분리하고 일시적 사용량을 연율화하지 않는다”는 정답 논리를 그대로 썼지만 choice: 1을 출력했다. 단순 오답과 판단-번호 매핑 실패를 분리해 집계해야 한다.

RAG

검색·컨텍스트 문제: 과거 복원 청크가 문항당 9K~11.6K 입력토큰을 만들었다. 원문이 길수록 정답 근거뿐 아니라 경쟁 신호도 함께 들어가 Main Prompt의 우선순위를 희석할 수 있다.

Persona

판별 문제: 일반 CFO가 객관식을 모두 맞혔고 Amy Hood Prompt는 H3·H5에서 오히려 직접 재무 지표를 먼저 보는 함정에 끌렸다. 프롬프트가 Amy Hood의 “무엇을 본다”보다 “어떤 순서로 본다”를 충분히 강제하지 못한다.

09 · Limitations

한계와 위협 요인

  • 반복성: 조건별 실행은 1회(n=1)다. temperature 0.2에서 실행 분산과 신뢰구간을 측정하지 않았다.
  • Before/After 혼입: 1.0.0과 2.0.0은 질문과 선택지가 달라 난이도 변화와 모델 변동을 분리할 수 없다.
  • 검색 방식: RAG는 기존 deterministic keyword top-1을 유지했다. 음수 lift를 다른 검색기나 RAG 구성 전체로 일반화할 수 없으며, 이 결과는 현재 검색·원문 구성에 한정된다.
  • 주관식 채점: 조건은 블라인드 처리했지만 단일 Codex 채점자이며 인간 다중 채점자 간 일치도를 측정하지 않았다.
  • 출력 오류: 선택 번호와 이유가 불일치하는 경우 현재 객관식 점수는 번호만 사용한다. 판단 논리 품질을 별도로 반영하지 않는다.
  • 판별 타당성: 일반 CFO의 12/12는 난이도만의 문제가 아니다. 정답이 일반적 재무 모범답안으로 식별 가능하다는 신호다.

10 · Next actions

개선 우선순위와 다음 실험 합격 기준

Priority 1

선택 번호–이유 정합성 검사

모델 이유가 어느 선택지의 판단 순서와 대응하는지 후처리로 비교하고 불일치 시 한 번만 재요청한다. 번호만으로 모델 판단을 오분류하는 오류를 먼저 제거한다.

합격: 반복 실행에서 번호–이유 불일치 0건
Priority 2

RAG 검색 적합성 평가

P1-P7별 top-1 청크가 정답 판단의 핵심 기준과 우선순위를 포함하는지 사람이 먼저 판정한다. 긴 원문 전체보다 해당 판단 근거가 있는 block 범위를 축소하는 실험을 추가한다.

합격: 검색 적합성 ≥6/7, RAG lift ≥0
Priority 3

Persona 판별 문항 재작성

일반 CFO도 동의하는 “좋은 답” 대신, 모두 재무적으로 합리적이지만 독립성·고객 신뢰·재무 한도를 적용하는 순서가 다른 선택지를 강화한다.

합격: Persona lift ≥+2, 일반 CFO 객관식 <80%
Priority 4

조건별 반복 실행

각 조건을 최소 3회 실행하고 평균·최솟값·최댓값·문항별 선택 일치율을 기록한다. 한 번의 샘플에 의존한 lift 방향을 재검증한다.

합격: 핵심 lift 방향이 모든 반복에서 동일
