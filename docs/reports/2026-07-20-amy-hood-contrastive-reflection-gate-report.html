Executive judgment

핵심 판단

현재 파이프라인은 구조적으로 유효하지만 의미적으로 느슨하다. 사건 ID가 다르다는 사실만으로 대조를 인정해 Gemma의 그럴듯한 일반화를 통과시켰다. 해결책은 모델 출력에 판단 축과 조건·행동 차이를 명시하게 하고, 코드 검증과 Codex 원문 검토를 연속 적용하는 것이다.

인력 재배치 대 OpenAI 투자통과거절두 사건 모두 저우선순위 자원을 줄이고 성장 분야에 집중한다. 반례가 아니라 같은 정책을 지지하는 보완 사례다.

결론: 자동 통과 3건을 승인했다면 구조화 메모리는 Amy Hood의 판단 기준이 아니라 Gemma의 사후적 서사를 저장했을 가능성이 높다.

Formal definition

동일 판단 축과 적격 대조

판단 축은 산업 분류가 아니라 경영진이 조정하는 대상, 비교 가능한 선택지, 선택을 바꾸는 관찰 조건의 조합이다. 도메인이 달라도 같은 판단 축이면 허용하지만, 분야가 다르다는 이유만으로 대조가 되지는 않는다.

판단 대상 + 비교 가능한 선택지 + 조건 변화 + 행동 변화 + 양쪽 원문 근거 = 적격 대조

판단 대상

자원 배분, 거래 구조, 가격·수익화처럼 실제로 조정하는 경영 변수.

전환 조건

수요, 성장 기회, 생산성, 통제 필요성처럼 공개자료에서 관찰 가능한 조건.

행동 차이

확대 대 축소, 인수 대 파트너십처럼 같은 선택지 집합 안에서 실질적으로 다른 행동.

Gemma output contract

강화된 구조화 JSON

Gemma 4는 판단 축과 대조 관계를 제안한다. JSON은 답변 형식이면서 자동 검사표다. 모델이 채운 내용을 그대로 신뢰하지 않는다.

Three-stage quality gate

생성에서 승인까지

01

Gemma 제안

봉인된 사건·증거만 사용해 강화 JSON을 생성한다.

02

코드 검증

필드, 행동 차이, 사건·증거 소유 관계, 홀드아웃을 검사한다.

03

Codex 검토

같은 판단 축인지, 인용문이 조건과 행동을 지지하는지 판정한다.

04

정책·릴리스

승인 성찰만 정책 유도에 사용하고 검증된 정책만 활성화한다.

지원·대조 행동은 같은 선택지 집합에 속하고 서로 달라야 한다.
양쪽 패턴에 조건, 사건 ID, 근거 스팬 ID가 각각 존재해야 한다.
패턴 스팬은 지정한 사건에 실제로 속해야 한다.
자동 통과 후에도 Codex 승인 전까지 review_required를 유지한다.

PoC execution gate

빠른 1회 평가를 위한 완료 조건

생성 한도

같은 봉인 입력으로 최대 2회 생성한다. 무한 재시도나 결과 선별 편향을 허용하지 않는다.

최소 승인

성찰 1개와 조건부 정책 1개가 자동·의미 검토를 모두 통과해야 한다.

평가 진입

활성 릴리스 해시와 홀드아웃 누출 0건을 확인한 뒤 4개 실험군 1회를 실행한다.

두 번의 생성 후에도 승인 성찰이 0개라면 모델 실패로 포장하지 않고 현재 5개 사건의 대조 데이터 부족으로 보고한다. 활성 릴리스 없이 RAG 실험을 강행하지 않는다.

본 문서는 공개자료 기반 비공식 Amy Hood Decision Advisor PoC의 개발자 품질 게이트 설계 보고서다. Amy Hood 본인이나 Microsoft의 공식 입장을 나타내지 않는다.
