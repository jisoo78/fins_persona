결론

원격 main의 Vector RAG·Cohere 재정렬·평가 자산은 수용하되, 기존의 정적 프롬프트 기반 평가 실행 경로는 B Track의 활성 불변 프롬프트 버전을 읽는 격리 모듈로 전환했다. A/B Track UI와 현재 평가 실행·채점·리포트 흐름은 유지했으며, 충돌 3건은 모두 해소되었다.

격리 · 제거 · 변경

격리

평가 기준선 전용 모듈

  • 원격 일반 RAG 평가기를 server/evaluation/decisionSimilarityBaseline.ts로 분리
  • 정적 프롬프트 자산은 런타임 소스가 아닌 과거 비교용 자료로 분류
  • 새 평가 결과에 활성 버전 ID와 SHA-256 해시 기록
제거 유지

폐기된 실행 경로

  • server/generateGeneralRagEvaluation.ts 복원 안 함
  • 기존 keyword/vector/train 평가 명령 복원 안 함
  • GraphRAG 및 정적 요약 의존 경로 재도입 안 함
  • EvaluationView의 정적 결과 import 재도입 안 함

운영 시 주의점

agent_prompts/prompts/amy-hood-main-system-prompt.md는 원격 실험 결과를 재현하기 위한 과거 정적 참고 자산이다. 현재 실행의 시스템 프롬프트는 UI에서 활성화한 불변 버전 저장소에서 가져오며, 서버가 이미 실행 중이라면 병합 후 코드를 반영하기 위해 재시작해야 한다.
이 보고서는 prompt/fins_persona에서 origin/main을 병합하면서 수행한 실제 충돌 해소와 검증 결과를 기준으로 작성되었다.
