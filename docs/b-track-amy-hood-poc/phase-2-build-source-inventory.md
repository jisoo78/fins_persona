# Phase 2: Source Inventory 만들기

## 목표

Amy Hood의 실제 재무 판단을 확인할 수 있는 공개자료의 목록을 만든다. 이 단계에서는 아직 페르소나 원칙을 만들지 않는다.

## Source Inventory란 무엇인가

조사에 사용할 문서의 주소, 날짜, 자료 유형, 관련 주제를 정리한 목록이다. 책의 목차를 먼저 만드는 것과 비슷하다.

## 실용적인 자료 선택 기준

다음 질문 중 두 개 이상에 “예”라고 답할 수 있는 자료를 우선한다.

- Amy Hood가 직접 말했는가?
- 실제 재무 문제나 선택 상황이 등장하는가?
- 선택의 이유나 고려한 위험이 설명되는가?
- 당시 상황과 날짜를 확인할 수 있는가?
- 원문 또는 전체 영상·transcript를 확인할 수 있는가?

어닝콜은 이 조건을 만족할 때만 사용한다. 단순 실적 숫자 낭독처럼 판단 과정이 없는 자료는 우선순위를 낮춘다.

## 우선 조사할 자료 유형

1. Amy Hood의 직접 인터뷰와 대담
2. CFO 관점의 콘퍼런스 발표와 Q&A
3. 인수, 투자, 비용 조정 등 특정 사건에 관한 공식 발언
4. 주주총회와 투자자 행사
5. 판단 근거가 포함된 어닝콜 Q&A
6. SEC 문서와 Microsoft 공식 문서
7. 신뢰할 수 있는 언론의 직접 인용 기사

## Source Inventory 예시

```json
{
  "source_id": "source_001",
  "title": "자료 제목",
  "url": "https://...",
  "published_at": "2025-01-01",
  "source_type": "interview",
  "source_grade": "primary",
  "speaker": "Amy Hood",
  "decision_domains": ["capital_allocation"],
  "has_full_text": true,
  "selection_reason": "투자 조건과 위험을 직접 설명함",
  "status": "selected"
}
```

## 만들어야 할 산출물

`data/b-track/amy-hood/source-inventory.json`

초보 단계에서는 데이터베이스를 만들 필요가 없다. JSON 파일 하나로 시작하고, PoC가 검증된 뒤 데이터베이스로 옮겨도 된다.

## Codex 요청 예시

> Phase 1의 범위를 기준으로 Amy Hood의 CFO 의사결정과 관련된 공개자료 후보를 찾아줘. 먼저 `source-inventory.json`만 작성하고 판단 원칙은 만들지 마. 각 자료의 선택 이유와 원문 여부를 표시해줘.

## 사람이 반드시 확인할 항목

- 링크가 실제 원문으로 연결되는가?
- Amy Hood가 직접 말한 자료인가?
- Microsoft 전체의 입장과 Amy Hood의 발언이 구분되는가?
- 단순 약력이나 실적 요약이 과도하게 포함되지 않았는가?

## 완료 기준

- 선택된 자료가 15~25개 있다.
- 다섯 개 재무 영역 중 최소 네 개가 포함된다.
- 각 자료에 URL, 날짜, 유형, 선택 이유가 있다.
- 직접 발언과 2차 보도가 구분되어 있다.
- 실제 판단 과정이 없는 자료는 제외되거나 낮은 우선순위로 표시되어 있다.
