# Phase 6: RAG 기반 Amy Hood 페르소나 평가

## 목표

생성된 `AMY_HOOD_PERSONA.*.md`를 같은 15문항으로 반복 평가하고, Gemma 4와 GPT-5-mini 결과를 실행 이력 단위로 비교한다. 평가 구조는 일반 RAG만 사용하며, 과거 복원 문항에만 승인된 공개 자료 청크를 최대 1개 제공한다.

## 평가 구성

| KPI | 문항 | 형식 | 모델 입력 |
| --- | ---: | --- | --- |
| 과거 복원 확인 | P1-P7, 7개 | 4지선다 | 메인 프롬프트 + 승인된 RAG 청크 최대 1개 |
| GitHub 인수 홀드아웃 | H1-H5, 5개 | 4지선다 | 메인 프롬프트만 사용, 2017-2019 자료와 정답 단서 제외 |
| 가상 시나리오 | S1-S3, 3개 | 주관식 | 메인 프롬프트만 사용 |

객관식은 정답 선택에 대해 문항당 1점으로 채점한다. 선택 이유는 사람이 판단 스타일을 검토할 수 있도록 보존하지만 점수에는 넣지 않는다. 주관식은 의사결정, 논리, 상충관계, 페르소나 일관성의 네 차원을 각각 0-2점으로 평가해 문항당 8점, 총 24점으로 채점한다.

## 안전 규칙

- 한 번의 모델 호출에는 한 문항만 전달한다.
- P 문항의 RAG 근거는 `source-inventory.json`에서 `selected`인 자료만 사용한다.
- H와 S 문항에는 RAG 청크를 주입하지 않는다.
- 문제 검토 화면에 보이는 정답, 함정 의도, 채점 기준은 생성 모델에 전달하지 않는다.
- 외부/Codex 주관식 채점 입력에는 답변 생성 provider와 model을 포함하지 않는다.
- 15문항이 모두 승인되기 전에는 평가 실행을 시작할 수 없다.
- 실행 결과는 문항마다 원자적으로 저장하므로 중단된 실행을 재개할 수 있다.
- OpenAI 실행은 Gemma 4 파이프라인 검증 게이트를 통과한 뒤에만 허용한다.

## Web UI 사용법

1. API 서버와 Web UI를 실행한다.

   ```bash
   npm run api
   npm run dev
   ```

2. 사이드바의 `평가 문항 검토`에서 15문항, 정답, 함정 선택지 의도와 주관식 루브릭을 검토한다.
3. 각 문항을 `승인` 또는 `수정 필요`로 저장한다. 15문항 모두 승인되어야 다음 단계로 갈 수 있다.
4. 사이드바의 `평가 비교`에서 기본값인 `Gemma 4 (local)`로 실행한다.
5. 중단된 실행은 같은 화면의 `재개` 버튼으로 이어서 수행한다.
6. S1-S3 답변은 외부 채점자 또는 Codex가 네 차원 점수와 채점 요약을 작성해 저장한다.
7. 완료된 두 실행을 선택해 질문별 답변, 객관식 점수, 주관식 점수, provider와 model을 비교한다.

유료 API 검증이 필요할 때만 실행 폼에서 `GPT-5-mini (OpenAI, 유료)`를 선택한다.

## CLI 사용법

문항 검토와 승인이 끝난 뒤 다음 명령으로 평가를 실행할 수 있다.

```bash
# 기본값: localhost:8080의 Gemma 4
npm run persona:evaluate

# Gemma 게이트 통과 후 유료 비교 실행
PERSONA_PROVIDER=openai npm run persona:evaluate
```

평가 회귀 테스트는 다음 명령으로 실행한다.

```bash
npm run evaluation:test
```

## 주요 산출물

- `evaluation/amy_hood_eval_questions.json`: 7/5/3 평가 질문
- `evaluation/amy_hood_eval_answer_key.json`: 객관식 정답·함정 의도와 주관식 루브릭
- `evaluation/amy_hood_eval_question_reviews.json`: 사람의 승인·수정 요청 상태
- `data/b-track/amy-hood/evaluation-runs/<run-id>.json`: 재개 가능한 실행 결과와 채점 기록
- `data/b-track/amy-hood/AMY_HOOD_PERSONA.gemma4.md`: 기본 평가 대상 프롬프트

## 반복 개선 절차

1. 현재 프롬프트로 Gemma 4 평가를 실행한다.
2. P/H 오답과 S 답변의 논리·상충관계 실패를 확인한다.
3. 공개 자료 분석 결과를 근거로 메인 프롬프트를 수정한다.
4. 동일한 승인 문항 세트로 새 실행을 만든다.
5. 이전 실행과 새 실행을 비교한다.
6. 로컬 파이프라인이 안정된 뒤 같은 문항 세트로 GPT-5-mini를 실행한다.

자료별 분석 청크 30개를 매 평가마다 다시 주입하지 않는다. 분석 청크는 메인 시스템 프롬프트를 만드는 입력이고, 평가 시에는 완성된 메인 프롬프트를 사용한다. 단, P1-P7만 장기 기억 검증을 위해 검색된 청크 1개를 추가한다.

## 완료 기준

- 평가 문항이 과거 복원 7개, 홀드아웃 5개, 가상 시나리오 3개로 고정되어 있다.
- 객관식 12개와 주관식 3개의 채점 방식이 실행 이력에 보존된다.
- H1-H5 실행 입력에서 2017-2019 홀드아웃 자료와 채점 단서가 차단된다.
- Gemma 4 실행이 15문항을 완료하고 중단 시 재개할 수 있다.
- 외부/Codex 주관식 채점 결과를 저장하고 두 완료 실행을 UI에서 비교할 수 있다.
- 지원되는 평가 코드와 UI는 구형 그래프 기반 평가 계약이나 정적 비교 결과에 의존하지 않는다.
