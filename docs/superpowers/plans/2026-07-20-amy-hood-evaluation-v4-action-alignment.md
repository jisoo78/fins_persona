# Amy Hood Evaluation v4 Action Alignment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and calibrate a frozen 30-scenario Evaluation v4 that measures Amy Hood policy-aligned action on a 1–10 Action Alignment Score across four experimental arms, while keeping Evaluation v3 immutable.

**Architecture:** Extend the existing reviewed Amy evidence → reflection → policy → memory release pipeline until one approved policy covers each of the five decision domains, then freeze an isolated external-CFO scenario dataset whose provenance never enters Amy memory. Candidate answers are generated by the existing local model and shared dynamic retriever, converted into arm-blind judge packets, graded with a mandatory reason-first two-stage judge, and reconciled into immutable JSON and HTML reports. V4 uses its own storage tree, routes, contracts, UI, and hashes; only the generic retrieval primitives are shared with v3.

**Tech Stack:** TypeScript 5.8, Node.js 22, `tsx --test`, Express 4, React 19, Vite 6, local OpenAI-compatible Gemma 4 E4B at `http://127.0.0.1:8080/v1`, local BGE-M3 embeddings at `http://127.0.0.1:8081/v1`, canonical JSON plus SHA-256, atomic filesystem persistence.

## Global Constraints

- Evaluation v3 questions, answer key, holdout manifest, runs, retrieval caches, reports, routes, and behavior are immutable; add regression tests before extracting any shared primitive.
- V4 contains exactly 30 open-ended scenarios: five domains × six scenarios, with exactly two `base_transfer`, two `boundary`, and two `reversal` scenarios per domain.
- The four arms are exactly `generic_cfo`, `amy_prompt`, `amy_policy_rag`, and `amy_full_rag`; supported repetition counts are exactly 1 and 5.
- A five-domain active Amy memory release is mandatory: at least one approved deployable policy per domain, at least two supporting events and one contrasting/reversal event per policy, and at least one direct Amy Hood evidence span per referenced event.
- Amy decision-time evidence and post-outcome evidence remain separated; Evaluation v3 holdout references remain forbidden from policy builds and memory indexes.
- External-CFO sources live only under `evaluation/v4/sources`; they must never enter `data/b-track/amy-hood/advisor`, the active Amy release, or the Amy memory index.
- Public generation input contains no external identity, historical action, eventual outcome, mapped policy ID, expected action, acceptable variant, identity conflict, or reference rationale.
- The local model uses port 8080 and a 16,384-token server context; BGE-M3 uses port 8081. V4 keeps the existing 6,000-token RAG-context and 12,000-token complete-request limits.
- RAG arms use query-dependent retrieval. Retrieval or context failure marks the answer/run incomplete and never falls back to prompt-only execution.
- Candidate responses are structured JSON with `action`, exactly three ordered `priorities`, `guardrails`, `reversalSignals`, and `rationale`; malformed output is retried once, then stored as failed.
- The judge is arm-blind and mandatory two-stage: stage one produces exactly one rationale sentence without seeing a numeric scale; stage two sees the frozen rationale and assigns an integer AAS from 1 through 10.
- The first live gate is 30 × 4 × 1 = 120 candidate answers. The remaining four repetitions run only if the frozen one-repetition benchmark passes its discrimination and judge-reliability gates.
- New or significantly modified test files start with a `Test Plan` comment containing one happy path, exactly three realistic edge cases by default, and failure-path coverage where persistence, model calls, or external sources are involved.
- All JSON mutations use atomic writes. Partial runs, partial grade batches, stale-hash reports, and partially built releases are never activated.
- No tool calling is introduced for the advisor or evaluation model; all selected memory is rendered into the model request.

  if (!Array.isArray(value) || value.length === 0
    || value.some((item) => typeof item !== 'string' || !item.trim())
    || (exact !== undefined && value.length !== exact)) {

export const parseEvaluationV4CandidateResponse = (text: string): EvaluationV4CandidateResponse => {
  let parsed: unknown;

  const record = parsed as Record<string, unknown>;
  const allowed = new Set(['action', 'priorities', 'guardrails', 'reversalSignals', 'rationale']);
  const unknown = Object.keys(record).find((key) => !allowed.has(key));
  if (unknown) throw new Error(`unknown candidate response field: ${unknown}`);
  if (typeof record.action !== 'string' || !record.action.trim()
    || typeof record.rationale !== 'string' || !record.rationale.trim()) {
    throw new Error('action and rationale are required');
  }
  const priorities = strings(record.priorities, 'priorities', 3) as [string, string, string];
  return {
    action: record.action.trim(), priorities,
    guardrails: strings(record.guardrails, 'guardrails'),
    reversalSignals: strings(record.reversalSignals, 'reversalSignals'),
    rationale: record.rationale.trim(),
  };

export const evaluatePolicyCoverage = ({ policies, events }: EvaluationV4PolicyCoverageInput): EvaluationV4PolicyCoverageReport => {
  const eventById = new Map(events.map((event) => [event.id, event]));
  const errors: string[] = [];
  const policyIdsByDomain: Partial<Record<DecisionDomain, string[]>> = {};
  for (const policy of policies) {
    const prefix = `policy ${policy.id}`;
    if (policy.status !== 'approved' || policy.policyKind !== 'deployable_policy' || !policy.review) errors.push(`${prefix} is not reviewed and deployable`);
    if (policy.schemaVersion !== 2 || !policy.guardrails?.length) errors.push(`${prefix} requires explicit guardrails`);
    if (new Set(policy.supportingEventIds).size < 2) errors.push(`${prefix} requires two supporting events`);
    if (new Set(policy.contrastingEventIds).size < 1) errors.push(`${prefix} requires a contrasting event`);
    const ids = [...new Set([...policy.supportingEventIds, ...policy.contrastingEventIds])];
    for (const id of ids) {
      const event = eventById.get(id);
      if (!event) errors.push(`${prefix} has unresolved event ${id}`);
      else if (event.status !== 'approved' || event.directAmyEvidenceIds.length === 0) errors.push(`${prefix} event ${id} requires direct Amy evidence`);
      else if (event.postOutcomeEvidenceIds.some((evidenceId) => policy.evidenceIds.includes(evidenceId))) errors.push(`${prefix} contains post-outcome evidence ${id}`);
    }
    if (!errors.some((error) => error.startsWith(prefix))) {
      policyIdsByDomain[policy.domain] = [...(policyIdsByDomain[policy.domain] ?? []), policy.id].sort();
    }
  }
  const coveredDomains = EVALUATION_V4_DOMAINS.filter((domain) => (policyIdsByDomain[domain]?.length ?? 0) > 0);
  const missingDomains = EVALUATION_V4_DOMAINS.filter((domain) => !coveredDomains.includes(domain));
  return { passed: errors.length === 0 && missingDomains.length === 0, coveredDomains, missingDomains, errors, policyIdsByDomain };
};
