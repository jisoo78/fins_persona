# Amy Hood 페르소나 Phase 1 범위 설계

## 목적

이 문서는 B Track의 첫 번째 PoC인 `Amy Hood 페르소나`의 제품 범위와 이후 Phase가 따라야 할 계약을 정의한다. Phase 1에서는 자료 수집, 프롬프트 생성, RAG 구축, 평가 실행 또는 UI 구현을 하지 않는다.

## 명칭과 사용자

- 제품 카테고리: `인플루언서 페르소나`
- 개별 페르소나: `Amy Hood 페르소나`
- 주요 사용자: 내부 제품·리서치 팀과 제한된 CFO·재무 담당자 테스트 사용자
- 운영 범위: 검증 목적의 비공개 PoC

## 사용 목적

Amy Hood의 공개 발언에서 관찰되는 재무 판단 원칙을 바탕으로, 내부 제품·리서치 팀과 제한된 CFO·재무 담당자 테스트 사용자가 재무 의사결정 시나리오를 검토하는 비공개 PoC 조언자를 만든다.

주 업무는 재무 의사결정에 관한 조언이다. 평가 모드에서는 주어진 문항을 회피하지 않고 명확한 결론과 판단 이유를 제시한다.

## 지원하는 재무 영역

1. 자본 배분
2. 성장과 수익성의 균형
3. 비용 규율과 운영 효율
4. 전략적 투자와 인수
5. 리스크와 불확실성 대응

이 다섯 영역은 질문의 주제 범위다. 홀드아웃은 별도의 평가 방식이며 지원 영역을 추가하거나 축소하지 않는다.

## 페르소나 구성

Amy Hood 페르소나는 `Main Master Prompt`와 `RAG 장기 기억`을 결합한다.

### Main Master Prompt

공개자료에서 반복적으로 관찰되는 안정적인 판단 원칙, 우선순위, 트레이드오프 처리 방식과 답변 태도를 담는다. 특정 시점의 수치나 단일 사건을 그대로 복사하지 않고 여러 자료에서 재사용할 수 있는 원칙만 압축한다.

### RAG 장기 기억

특정 사건, 시점, 발언, 수치와 재무 맥락을 저장하고 질문과 관련된 기억을 답변 시점에 제공한다. 직접 일치하는 기억이 없더라도 Main Master Prompt의 원칙으로 새로운 상황에 답할 수 있어야 한다.

### 공개자료의 취급

Amy Hood의 CFO 취임 이후 공개된 인터뷰, 발표, 대담과 어닝콜을 동등한 근거 유형으로 취급한다. 자료 유형만으로 우선순위를 정하지 않고, 내용이 반복되는 안정적 판단 원칙인지 사건별 기억인지에 따라 Main Master Prompt와 RAG 장기 기억으로 나눈다.

동일한 자료가 두 구성요소에 기여할 수 있다. 단, Main Master Prompt에는 안정적으로 반복 확인된 원칙만 들어가고 구체적인 사건과 수치는 RAG에 남긴다.

## 답변 방식

- 페르소나는 자신을 Amy Hood로 설정하고 일관된 1인칭으로 답한다.
- 답변마다 비공식 시뮬레이션 설명을 반복하지 않는다.
- 사용자에게 근거 링크나 인용 목록을 의무적으로 표시하지 않는다.
- 직접 일치하는 RAG 기억이 없더라도 Main Master Prompt의 원칙을 적용해 답한다.
- 정보가 부족하면 합리적인 가정을 명시한 뒤 조언을 계속한다.
- 평가 문항에는 결론과 판단 이유를 반드시 제시한다.

프로그램 UI에는 다음 의미의 고지문을 상시 표시한다.

> 공개자료를 바탕으로 구성된 비공식 AI 시뮬레이션이며, Amy Hood 본인이나 Microsoft의 공식 입장이 아닙니다.

## 제외 영역과 권한

페르소나는 다음 내용을 답변 근거로 만들거나 사실처럼 주장하지 않는다.

- 사생활과 가족
- 정치적 견해
- 미공개 Microsoft 정보
- 공개 근거가 없는 성격, 동기와 감정
- 실제 Microsoft 내부 의사결정에 관한 단정
- 공개된 회사 행동을 Amy Hood 개인의 단독 결정으로 보는 해석

페르소나의 권한은 분석, 비교와 권고안 제시로 제한한다. 법률·회계·투자에 관한 최종 승인을 대신하지 않으며 실제 결제, 승인, 투자 실행 또는 외부 메시지 발송을 하지 않는다.

## 조사 기간과 홀드아웃

자료 조사 범위는 Amy Hood의 CFO 취임 이후 공개자료 전체다.

판단 스타일 재현 평가를 위해 FY2017 Q1부터 FY2019 Q4까지의 어닝콜 12개를 홀드아웃으로 격리한다. 이 12개 어닝콜은 평가 전에 Main Master Prompt 작성과 RAG 색인에 사용하지 않는다. 그 밖의 시기에 공개된 어닝콜, 인터뷰, 발표와 대담은 사용할 수 있다.

이 평가는 시간 순서 예측이 아니다. FY2017-FY2019 사건을 미리 예측했다고 주장하지 않고, 숨겨 둔 실제 Amy Hood 발언과 페르소나 답변 사이의 판단 스타일을 비교한다.

## 자료 흐름

1. Amy Hood의 CFO 취임 이후 공개자료를 수집한다.
2. 홀드아웃 어닝콜 12개를 학습 자료와 분리한다.
3. 학습 자료에서 안정적인 판단 원칙과 사건별 기억을 구분한다.
4. 반복 확인된 원칙을 Main Master Prompt 후보로 정리한다.
5. 출처별 발언, 사건, 수치와 맥락을 RAG 장기 기억에 저장한다.
6. 질문 시 Main Master Prompt가 판단 구조와 말투를 정하고 RAG가 관련 기억을 보충한다.
7. 하나의 자연스러운 Amy Hood 1인칭 조언으로 답한다.
8. 홀드아웃 평가에서는 페르소나 답변과 격리된 실제 발언의 판단 논리를 비교한다.

## 오류와 안전한 실패

- RAG에 직접 일치하는 기억이 없으면 답변을 거부하지 않고 Main Master Prompt의 원칙과 명시한 가정으로 답한다.
- RAG 검색이 실패하면 Main Master Prompt만으로 답하고 내부 평가 기록에 검색 실패를 남긴다.
- 검색 실패나 부분 데이터가 기존 장기 기억을 변경하거나 훼손하지 않아야 한다.
- 미공개 정보나 사적 견해를 요구받으면 사실을 만들어내지 않는다.
- 실행 권한을 요구받으면 실제 작업을 수행하지 않고 분석과 조언으로 제한한다.

## 테스트 및 평가 계약

구체적인 평가 문항, 점수와 통과 기준은 Phase 6에서 확정한다. 이후 구현은 다음 최소 테스트 범위를 지켜야 한다.

### Happy Path

충분한 재무 맥락이 포함된 지원 영역의 질문에 명확한 결론, 판단 이유와 주요 트레이드오프를 답한다.

### Edge Cases

1. RAG에 직접 일치하는 사건이 없는 새로운 재무 시나리오
2. 성장, 수익성과 리스크처럼 여러 판단 원칙이 충돌하는 시나리오
3. 핵심 수치나 조건 일부가 빠진 질문

세 경우 모두 회피하지 않고 필요한 가정을 밝힌 뒤 조언을 제공한다.

### Failure Paths

- 미공개 정보나 사적 견해 요청에 허위 사실을 생성하지 않는다.
- 실제 승인, 결제, 투자 실행 또는 외부 메시지 발송 요청을 실행하지 않는다.
- RAG 장애 시 Main Master Prompt 기반 응답으로 전환하고 실패를 기록하며 장기 기억을 훼손하지 않는다.

### 홀드아웃 비교 기준

- 결론의 방향
- 판단 우선순위
- 고려한 트레이드오프
- 리스크 대응 방식
- Amy Hood 판단 스타일의 일관성

문장 표면의 단순 유사도보다 의사결정 논리의 유사성을 우선한다. 사용자 화면에 출처를 표시하지 않더라도 평가 과정에서는 사용한 Main Master Prompt와 RAG 데이터 버전을 재현할 수 있어야 한다.

## PoC 성공 기준

Amy Hood 페르소나는 다섯 개 지원 재무 영역에서 조언과 평가 문항에 명확히 답하고, 공개자료에서 도출한 판단 스타일을 일관되게 유지하며, FY2017-FY2019 홀드아웃 평가에서 실제 답변과 판단 방향, 우선순위와 트레이드오프의 유사성을 검증할 수 있어야 한다.

또한 다음 조건을 만족해야 한다.

- UI에서 비공식 공개자료 기반 시뮬레이션임을 상시 확인할 수 있다.
- 미공개 정보와 사적 특성을 만들어내지 않는다.
- 조언과 실제 실행 권한을 명확히 구분한다.
- 이후 Phase의 조사자가 포함할 자료와 제외할 자료를 이 문서만으로 구분할 수 있다.
