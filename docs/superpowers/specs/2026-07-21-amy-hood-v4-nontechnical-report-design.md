# Amy Hood Evaluation V4 비전문가용 상세 보고서 설계

## 목적

기존 Evaluation V4 실험 결과를 프로젝트를 처음 접하는 비개발자도 이해할 수 있는 HTML 보고서로 다시 구성한다. 보고서는 기술 구현을 설명하는 데 그치지 않고, 무엇을 검증했고 결과가 무엇을 의미하며 어디까지 믿을 수 있는지를 쉬운 한국어로 전달해야 한다.

## 대상 독자와 표기 원칙

- 대상 독자는 개발 경험이 없는 한국인 의사결정자다.
- 한국어를 기본 언어로 사용한다.
- 필요한 영어 용어는 한글을 먼저 쓰고 영어를 병기한다. 예: `행동 정합성 점수(Action Alignment Score, AAS)`.
- `RAG`, `Prompt`, `Full RAG` 같은 코드명은 단독으로 사용하지 않고 최초 등장 시 쉬운 뜻을 설명한다.
- 기술 식별자, 해시, 실행 ID는 본문 흐름을 방해하지 않도록 마지막 재현 정보에서만 제공한다.
- 점수 차이는 “개선됐다”로 끝내지 않고 비교 기준과 절대 크기를 함께 설명한다.

## 보고서 구조

1. **한눈에 보는 결론**
   - 이번 실험의 질문, 핵심 수치, 현재 판정을 세 문장 안에서 제시한다.
   - `GO`가 제품 배포 승인이 아니라 다음 확대 실험 진행 판정임을 명시한다.

2. **프로젝트 소개**
   - Amy Hood Decision Advisor가 공개 자료에서 판단 기준을 추출해 새로운 CFO 상황에 적용하는 시스템임을 설명한다.
   - Amy Hood 본인 또는 Microsoft 공식 입장이 아닌 비공식 AI 시뮬레이션임을 표시한다.

3. **실험 목적과 가설**
   - Main Prompt만으로 차이가 생기는지, 구조화 정책 검색이 추가 행동 차이를 만드는지, 사건·근거까지 포함한 검색이 더 나은지를 검증한다.
   - “Amy Hood 완전 복제”가 아니라 “Prompt/RAG가 답변 행동에 방향성 있는 차이를 만드는가”가 이번 교정 실험의 범위임을 구분한다.

4. **네 가지 실험군**
   - 일반 CFO 조언자(Generic CFO)
   - Amy Hood 메인 프롬프트(Amy Prompt)
   - Amy Hood 프롬프트 + 정책 검색(Policy RAG)
   - Amy Hood 프롬프트 + 정책·사건·근거 검색(Full RAG)
   - 각 실험군에서 모델에게 추가되는 정보와 비교 목적을 쉬운 표로 제시한다.

5. **평가 자료와 절차**
   - 5개 의사결정 영역, 영역별 2문항, 총 10개 시나리오를 설명한다.
   - 동일 모델과 동일 질문을 네 조건에 적용해 총 40개 답변을 얻는 과정을 단계별로 보여준다.
   - 채점 전 실험군·모델·실행 ID를 숨긴 블라인드 채점(Blind Judging)을 설명한다.

6. **평가 지표**
   - 행동 정합성 점수(AAS) 1~10점의 의미를 구간별로 설명한다.
   - 행동, 우선순위, 안전장치, 판단 반전 신호의 네 채점 축을 사례와 함께 설명한다.
   - 평균 점수, 비교 향상 폭(Lift), 영역별 점수, 검색 정확도, 행동 변화 문항 수를 각각 쉬운 문장으로 정의한다.

7. **정량 결과와 해석**
   - 전체 평균, 비교 향상 폭, 영역별 결과를 표로 제시한다.
   - Main Prompt 단독은 일반 CFO보다 개선되지 않았고, 동적 RAG 조건에서 개선이 나타났다는 점을 강조한다.
   - 정책 검색과 Full RAG의 차이가 작다는 점도 과장 없이 설명한다.

8. **잘된 점과 취약점**
   - 검색된 정책 영역, 근거 첨부, 캐시 일치가 모두 100%였음을 설명한다.
   - 주주환원·재무위험 영역이 가장 낮고, 반전 시나리오가 기본 이전 시나리오보다 어렵다는 점을 명시한다.

9. **근거 완전성과 한계**
   - 공식 1차 자료 10개, 독립 2차 자료 0개, 사후 결과 근거 0개를 숨기지 않는다.
   - 단일 모델, 단일 채점자, 1회 반복, 10문항이므로 통계적 안정성과 일반화를 입증하지 못했음을 설명한다.

10. **최종 판단과 다음 단계**
    - 30문항·5회 반복으로 확대할 조건부 GO로 결론 낸다.
    - 주주환원 정책과 반례 보강, 독립 2차 자료 확보, 복수 채점자 비교를 우선 과제로 제시한다.

11. **재현 정보**
    - 모델, 포트, 실험 그룹 ID, 시나리오 해시, 응답·채점 수를 별도 표로 제공한다.

## 시각 표현

- 문서 상단에는 핵심 결론과 네 실험군 평균을 카드 형태로 배치한다.
- 색상은 상승·경고·한계를 구분하는 보조 수단으로만 사용하고, 모든 의미를 텍스트로도 표시한다.
- 표 머리글과 수치 옆에 쉬운 해설을 제공한다.
- 모바일에서도 가로 스크롤이나 표 붕괴 없이 읽을 수 있도록 반응형 스타일을 적용한다.

## 생성 방식

- 기존 정적 HTML만 직접 수정하지 않는다.
- `server/evaluationV4/report.ts`의 HTML 생성기를 개정하고 동일한 실험 그룹으로 보고서를 재생성한다.
- 실험 점수, 검색 지표, 영역별 결과는 기존 정식 JSON 보고서에서 읽으며 수치를 수동 복사하지 않는다.
- 보고서 생성 테스트에 주요 섹션, 영·한 병기, 비복제 경고와 재현 정보가 포함되는지 검증한다.

## 완료 기준

- 비전문가가 프로젝트 목적, 네 실험군 차이, AAS 의미, 핵심 결과와 한계를 순서대로 이해할 수 있다.
- 영어 전문 용어가 한글 설명 없이 단독으로 노출되지 않는다.
- 40개 답변과 40개 블라인드 채점, 네 실험군 점수 및 비교 향상 폭이 정확히 표시된다.
- `GO`가 다음 실험 진행 판정임을 명확히 구분한다.
- 기존 Evaluation V4 테스트, TypeScript 검사와 production build가 통과한다.
