# 평가 실행 ID 복사 기능 설계

## 목적

사용자가 평가 실행 ID 전체를 Codex 채점 요청이나 API 조회에 바로 사용할 수 있도록, 평가 화면의 현재 실행과 비교 대상 실행에서 UUID를 한 번에 복사한다. 화면에 표시된 앞 8자리 축약값이 아니라 항상 전체 `runId`를 클립보드에 기록한다.

## 범위

### 현재 실행 요약

`EvaluationRunSummary`의 전체 실행 ID 옆에 `ID 복사` 버튼을 둔다. 버튼은 현재 실행의 `run.runId`를 복사한다.

### 평가 이력 비교

`EvaluationRunHistory`의 좌·우 실행 선택 영역에서 각 선택값에 대응하는 `ID 복사` 버튼을 제공한다. 실행이 선택되지 않은 쪽의 버튼은 비활성화한다. 드롭다운의 축약 표시는 유지하되 복사값은 전체 UUID를 사용한다.

## 상호작용

- 기본 라벨은 `ID 복사`다.
- 복사 성공 후 해당 버튼만 2초 동안 `복사됨`으로 표시한다.
- 복사 실패 후 해당 버튼만 2초 동안 `복사 실패`로 표시한다.
- 좌·우 또는 현재 실행 버튼의 상태는 서로 독립적이다.
- 빠르게 다시 클릭하면 마지막 클릭을 기준으로 상태 복귀 시간을 갱신한다.

## 복사 경계

공용 복사 유틸리티는 다음 순서로 동작한다.

1. `navigator.clipboard.writeText`를 사용할 수 있으면 전체 실행 ID를 전달한다.
2. Clipboard API가 없으면 임시 텍스트 요소를 선택하고 `document.execCommand('copy')`를 호출하는 폴백을 사용한다.
3. 기본 경로와 폴백이 모두 실패하면 실패 결과를 반환하며 예외를 UI 바깥으로 전파하지 않는다.

버튼 컴포넌트는 복사 결과와 일시적인 라벨 상태만 담당하고, 실행 데이터 조회나 저장에는 관여하지 않는다.

## 구성 요소

- `copyTextToClipboard`: Clipboard API와 폴백을 캡슐화하는 공용 유틸리티
- `CopyRunIdButton`: 전체 실행 ID, 비활성화 여부와 독립적인 피드백 상태를 관리하는 버튼
- `EvaluationRunSummary`: 현재 실행 버튼 배치
- `EvaluationRunHistory`: 선택된 좌·우 실행 버튼 배치

## 오류 처리

- 복사 실패는 평가 실행이나 비교 상태에 영향을 주지 않는다.
- Clipboard API 거부와 폴백 실패는 버튼의 `복사 실패` 상태로만 알린다.
- 실행이 선택되지 않았거나 ID가 빈 경우 복사를 시도하지 않는다.

## 테스트 계획

### Happy Path

- 현재 실행의 전체 UUID가 Clipboard API로 복사되고 성공 상태가 표시된다.

### Edge Cases

1. Clipboard API가 없으면 폴백으로 전체 UUID를 복사한다.
2. 좌·우 복사 버튼의 피드백 상태가 서로 영향을 주지 않는다.
3. 같은 버튼을 빠르게 다시 클릭하면 마지막 클릭 이후 2초 동안 성공 상태가 유지된다.

### Failure Path

- Clipboard API와 폴백이 모두 실패하면 예외를 전파하지 않고 `복사 실패` 상태를 표시한다.
- 실행이 선택되지 않은 비교 버튼은 비활성화되어 복사를 시도하지 않는다.

## 완료 기준

- 현재 실행과 비교 대상 좌·우 실행에서 전체 UUID를 복사할 수 있다.
- 성공·실패 피드백이 해당 버튼에만 표시된다.
- 클립보드 미지원 및 거부 상황에서 안전하게 폴백하거나 실패를 알린다.
- 기존 평가 실행, 채점, 비교 기능의 테스트와 빌드가 계속 통과한다.
