# Amy Hood Evaluation Hardening 1.5.0

Date: 2026-07-20

## 목적

1.4.0 평가에서는 Gemma 4 12B, Gemma 4 E4B, Phi-4-mini가 모두 60/60을 기록했다. 따라서 현재 문항이 모델에게 너무 예측 가능한 형태일 수 있다고 보고, 1.5.0에서는 모델 차이가 드러날 수 있도록 함정 선택지를 더 정교하게 수정했다.

## 변경 내용

- 평가셋 버전을 1.4.0에서 1.5.0으로 변경했다.
- 총 문항 수는 60문항으로 유지했다.
- 문항 구성은 유지했다.
  - 과거 기억 복원: 20문항
  - M&A/GitHub 판단: 20문항
  - 가상 시나리오 객관식: 20문항
- 과거 기억 복원 문항은 사실 확인 성격이 강해 유지했다.
- GitHub 홀드아웃 20문항과 가상 시나리오 20문항의 오답 선택지를 강화했다.

## 함정 선택지 수정 방식

기존 오답은 정답과 비교했을 때 상대적으로 명확하게 틀린 선택지가 많았다.

1.5.0에서는 오답도 Amy Hood식 판단처럼 보이도록 아래 요소를 일부 포함했다.

- 장기 플랫폼 가치
- 고객 신뢰
- 개발자 생태계
- 비용 규율
- 수익화 가능성
- 투자자 커뮤니케이션

다만 각 오답은 핵심 기준 하나를 빠뜨리도록 구성했다.

- 재무 영향은 사후 관리로 미룸
- 단기 수익화 또는 비용 방어를 우선함
- 성과 판단을 단일 지표로 단순화함
- 커뮤니티 신뢰, 현금흐름, 통합비용, capacity 유연성 중 일부를 누락함

## 기대 효과

이번 변경은 단순 키워드 매칭으로 정답을 고르기 어렵게 만드는 것이 목적이다. 모델이 Amy Hood의 의사결정 기준을 실제로 적용하는지, 아니면 장기 성장·고객 가치 같은 표면적 표현만 따라가는지 더 잘 확인할 수 있다.

## 검증

- `npm run lint` 통과
- `npm run evaluation:test` 통과
- 평가 번들 버전, 문항 수, 정답 키, 리뷰 파일 모두 1.5.0으로 정합성 확인
