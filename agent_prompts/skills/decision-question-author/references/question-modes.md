# Question Modes

Use `stage` to define the purpose of a question. Use `question_mode` to define how the question is expressed.

## direct_choice

Use for simple preferences or format choices where trade-offs are not the main signal.

Good for:

- Communication style
- Simple default preference
- Clear mutually exclusive behaviors

Avoid when the answer would become generic or socially desirable.

## scenario_choice

Use as the default mode for most pre-interview questions. Present a realistic situation and ask what the user would do.

Good for:

- `preference`
- `context_shift`
- `core_value`

The scenario should include enough context to force judgment, but not so many variables that the user has to solve a puzzle.

## attribute_tradeoff

Use only when a user's decision criteria are best exposed by comparing multiple plausible options with different attribute combinations.

Good for:

- `preference`, when default criteria are otherwise too abstract
- `priority_order`, when conflicts among criteria are the core signal

Use sparingly:

- Recommend at most 2 per category.
- Avoid consecutive use.
- Avoid when one scenario sentence can reveal the same criterion.

## boundary_check

Use when the stage asks for a stop, reject, defer, or escalation line.

Good for:

- `red_line`
- Some `core_value` questions

Ask for the threshold, condition, or situation where the decision changes.

## priority_order

Use when two or more values collide and the user must choose what wins first.

Good for:

- `priority_order`
- Some `context_shift` questions

Make the conflict explicit. Do not let the user choose "both" unless the product explicitly supports ranking or weighted answers.
