# Amy Hood 어려운 평가 질문 15문항

## 과거 기억 복원

1. Amy Hood가 cloud와 AI capacity 투자를 설명할 때, 매출 성장, gross margin, operating margin 사이의 연결 구조를 어떻게 설명했는가?

2. 대규모 CapEx와 free cash flow가 동시에 언급되는 상황에서 Amy Hood는 투자자에게 어떤 재무적 긴장을 설명했는가?

3. 사업부별 가이던스를 설명할 때 Amy Hood가 Azure, Office, LinkedIn, FX, commercial bookings를 어떤 판단 축으로 나누어 설명했는가?

4. Microsoft의 인수 관련 설명에서 Amy Hood는 단기 비용 부담, 통합 리스크, 장기 성장 기여를 어떤 순서와 기준으로 판단했는가?

5. commercial cloud 전환 과정에서 Amy Hood가 반복적으로 강조한 성장 신호와 수익성 개선 경로는 무엇인가?

## 미래 예측

6. Microsoft가 GitHub 인수를 검토하고 있다. GitHub는 개발자 생태계와 네트워크 효과는 강하지만 단기 수익성은 낮고 인수 가격 부담이 크다. Amy Hood 페르소나라면 인수 승인 여부를 어떤 기준으로 결정할까?

7. GitHub 인수 후 Microsoft가 제품을 빠르게 통합하면 비용 시너지는 생기지만 개발자 커뮤니티 신뢰가 훼손될 수 있다. Amy Hood 페르소나라면 이 리스크가 인수 여부에 어떤 영향을 준다고 판단할까?

8. GitHub는 인수 당시 직접 매출 기여가 제한적이지만 Azure, Visual Studio, DevOps, 보안 제품과 연결될 수 있다. Amy Hood 페르소나라면 인수 찬반 판단을 위해 어떤 수익화 경로를 확인하려 할까?

9. GitHub 인수 가격이 높아 단기 EPS와 operating margin에 부담이 생길 수 있다. Amy Hood 페르소나라면 어느 정도의 단기 희석을 감수하고, 어떤 회수 논리가 있어야 인수를 승인할까?

10. 위 조건을 종합했을 때 Amy Hood 페르소나라면 GitHub 인수를 승인할까, 보류할까, 거절할까? 결론과 함께 반드시 확인해야 할 재무 조건과 비재무 조건을 제시하라.

## 의사결정 유사도

11. 인수 대상의 핵심 자산이 커뮤니티 신뢰인데, 완전 통합하면 비용 절감은 가능하지만 사용자 이탈 위험이 크다. Amy Hood식 의사결정이라면 어떤 레드라인을 둘까?

12. AI capacity가 부족해 first-party 서비스, 전략 고객, 단기 매출 기회 중 일부만 지원할 수 있다. Amy Hood식 판단이라면 어떤 배분 원칙을 세울까?

13. 경기 둔화로 비용 절감 압박이 커졌지만 AI와 클라우드 R&D는 장기 성장에 중요하다. Amy Hood식 의사결정이라면 어떤 예산은 줄이고 어떤 예산은 유지할까?

14. 성장률은 높지만 capacity 제약, 비용 압박, FX 리스크가 동시에 존재한다. Amy Hood 페르소나라면 투자자에게 어떤 순서와 톤으로 설명할까?

15. 위 데이터로 만든 Amy Hood CFO Advisor가 새로운 재무 의사결정 안건을 검토할 때 반드시 지켜야 할 핵심 지침 7가지를 작성하라.
