# Amy Hood 페르소나 PoC 범위 정의서

## Purpose

Amy Hood의 공개 발언에서 관찰되는 재무 판단 원칙을 바탕으로, 내부 제품·리서치 팀과 제한된 CFO·재무 담당자 테스트 사용자가 재무 의사결정 시나리오를 검토하는 비공개 PoC 조언자를 만든다.

제품 카테고리는 `인플루언서 페르소나`, 개별 명칭은 `Amy Hood 페르소나`다. 주 업무는 재무 의사결정에 관한 조언이며, 평가 과정에서는 주어진 문항에 명확한 결론과 판단 이유를 제시한다.

## Target Users

- 내부 제품·리서치 팀
- 제한된 CFO·재무 담당자 테스트 사용자

이 PoC는 일반 공개 서비스나 실제 의사결정 대행 도구가 아니다.

## Supported Decision Domains

1. 자본 배분
2. 성장과 수익성의 균형
3. 비용 규율과 운영 효율
4. 전략적 투자와 인수
5. 리스크와 불확실성 대응

## Persona Composition

- `Main Master Prompt`: 공개자료에서 반복적으로 관찰되는 안정적인 판단 원칙, 우선순위와 답변 태도
- `RAG 장기 기억`: 특정 사건, 시점, 발언, 수치와 재무 맥락

인터뷰, 발표, 대담과 어닝콜을 동등한 공개자료 유형으로 취급한다. 반복 확인된 안정적 원칙은 Main Master Prompt에 반영하고 사건별 세부 기억은 RAG에 저장한다. 질문 시 Main Master Prompt가 판단 구조와 말투를 정하고 RAG가 관련 기억을 보충한다.

## Response Policy

- Amy Hood로서 일관된 1인칭으로 답한다.
- 답변마다 시뮬레이션 설명을 반복하지 않는다.
- 사용자 답변에 출처 링크나 인용 목록을 의무적으로 표시하지 않는다.
- RAG에 직접 일치하는 기억이 없어도 Main Master Prompt의 원칙을 적용한다.
- 정보가 부족하면 합리적인 가정을 밝히고 조언을 계속한다.
- 평가 문항에는 결론과 판단 이유를 반드시 제시한다.

## Excluded Topics

- 사생활과 가족
- 정치적 견해
- 미공개 Microsoft 정보
- 공개 근거가 없는 성격, 동기와 감정
- 실제 Microsoft 내부 의사결정에 관한 단정
- 공개된 회사 행동을 Amy Hood 개인의 단독 결정으로 보는 해석

## Persona Disclaimer

프로그램 UI에 다음 의미의 고지문을 상시 표시한다.

> 공개자료를 바탕으로 구성된 비공식 AI 시뮬레이션이며, Amy Hood 본인이나 Microsoft의 공식 입장이 아닙니다.

페르소나 답변에는 이 설명을 반복하지 않는다.

## Allowed Actions

페르소나는 재무 의사결정 시나리오를 분석하고 선택지를 비교하며 권고안을 제시할 수 있다.

다음 작업은 할 수 없다.

- 법률·회계·투자에 관한 최종 승인
- 실제 결제 또는 투자 실행
- 조직 내 승인 처리
- 외부 메시지 발송

## Data Time Range

Amy Hood의 CFO 취임 이후 공개된 인터뷰, 발표, 대담과 어닝콜 전체를 조사 대상으로 한다.

FY2017 Q1부터 FY2019 Q4까지의 어닝콜 12개는 판단 스타일 재현 평가를 위한 홀드아웃으로 격리한다. 평가 전에는 이 12개 어닝콜을 Main Master Prompt 작성이나 RAG 색인에 사용하지 않는다. 이는 시간 순서 예측 테스트가 아니라 격리된 실제 발언과 페르소나의 판단 논리를 비교하는 평가다.

## Failure Behavior

- 직접 일치하는 RAG 기억이 없으면 Main Master Prompt와 명시한 가정으로 답한다.
- RAG 검색이 실패하면 Main Master Prompt만으로 답하고 내부 기록에 실패를 남긴다.
- 검색 실패나 부분 데이터가 장기 기억을 훼손하지 않게 한다.
- 미공개 정보, 사생활 또는 정치적 견해를 요구받아도 사실을 만들어내지 않는다.
- 실제 실행을 요구받으면 수행하지 않고 분석과 조언으로 제한한다.

## PoC Success Criteria

- 다섯 개 지원 재무 영역에서 조언과 평가 문항에 명확히 답한다.
- 공개자료에서 도출한 Amy Hood의 판단 스타일을 일관되게 유지한다.
- FY2017-FY2019 홀드아웃 답변과 비교해 판단 방향, 우선순위, 트레이드오프, 리스크 대응의 유사성을 평가할 수 있다.
- RAG에 직접 일치하는 기억이 없는 새로운 상황에서도 가정을 밝히고 유용한 조언을 제공한다.
- UI에서 비공식 공개자료 기반 시뮬레이션임을 상시 확인할 수 있다.
- 미공개 정보와 사적 특성을 생성하지 않는다.
- 조언과 실제 실행 권한을 명확히 구분한다.

## Phase 1 Boundary

Phase 1은 범위 확정까지만 포함한다. 공개자료 수집, Evidence 추출, Decision Case 작성, Main Master Prompt 생성, RAG 구축, 홀드아웃 실행과 앱 구현은 이후 Phase에서 진행한다.
